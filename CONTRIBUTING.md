# Contributing to HorseClaw

Thank you for your interest in contributing to HorseClaw! This document provides guidelines and instructions for contributing.

## 🌐 Languages

This project is bilingual (English/中文). Contributions in either language are welcome.

## 🚀 Getting Started

1. Fork the repository
2. Clone your fork: `git clone https://github.com/YOUR_USERNAME/horseclaw.git`
3. Install development dependencies: `make install-dev`
4. Create a branch: `git checkout -b feature/your-feature`

## 📝 Development Workflow

### Code Style

- Follow PEP 8
- Use Black for formatting: `make format`
- Maximum line length: 100 characters
- Add type hints for new functions

### Testing

- Write tests for new features
- Ensure all tests pass: `make test`
- Maintain coverage: `make test-cov`

### Documentation

- Update README.md for user-facing changes
- Add docstrings to new functions
- Update CHANGELOG.md

## 🔍 Pull Request Process

1. Update documentation
2. Add tests
3. Ensure CI passes
4. Request review from maintainers

## 📋 Code Review Criteria

- Code quality and style
- Test coverage
- Documentation completeness
- Bilingual support (if applicable)

## 🐛 Reporting Bugs

Use GitHub Issues with:
- Clear title and description
- Steps to reproduce
- Expected vs actual behavior
- Python version and environment

## 💡 Feature Requests

Open an issue describing:
- Use case
- Proposed solution
- Alternative solutions considered

## 📄 License

By contributing, you agree that your contributions will be licensed under the MIT License.

# 为 HorseClaw 做贡献

感谢您有兴趣为 HorseClaw 做贡献！本文档提供了贡献的指南和说明。

## 🚀 开始

1. Fork 仓库
2. 克隆您的 fork: `git clone https://github.com/您的用户名/horseclaw.git`
3. 安装开发依赖: `make install-dev`
4. 创建分支: `git checkout -b feature/您的功能`

## 📝 开发工作流程

### 代码风格

- 遵循 PEP 8
- 使用 Black 格式化: `make format`
- 最大行长度: 100 字符
- 为新函数添加类型提示

### 测试

- 为新功能编写测试
- 确保所有测试通过: `make test`
- 保持覆盖率: `make test-cov`

### 文档

- 为用户可见的更改更新 README.md
- 为新函数添加文档字符串
- 更新 CHANGELOG.md

## 🔍 拉取请求流程

1. 更新文档
2. 添加测试
3. 确保 CI 通过
4. 请求维护者审查

## 🐛 报告错误

使用 GitHub Issues，包含：
- 清晰的标题和描述
- 重现步骤
- 预期与实际行为
- Python 版本和环境

## 📄 许可证

通过贡献，您同意您的贡献将在 MIT 许可证下许可。
